# Charters

**Project-level binding documents.** One per project — defines classification, personas active, UNIVERSAL bindings, and escalation rules.

A charter answers: "what kind of project is this, which personas staff it, and which Factory artifacts apply?"

The companion machine-readable form is `factory.json` at the repo root.

## Format
- Frontmatter: `name`, `description`, `type: charter`, `date`, `status`, `project`
- Body: identity, classification, personas, factory bindings, escalation
